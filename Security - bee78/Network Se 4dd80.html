Network Security

Wifi

This guide is for anyone who cannot use a hardwire connection. Do not use Wi-Fi unless you have no other option.

  • do not hide your SSID (see:Â https://www.howtogeek.com/howto/28653/debunking-myths-is-hiding-your-wireless-ssid-really-more-secure/)
    • set an innocuous SSID like "family home wifi" and not "bitcoin treasure trove"
    • do not use a name that identifies your ISP, like "Comcast Wifi"
    • only allow certain MAC addresses to connect to your router
      • Linksys -Â https://www.linksys.com/us/support-article?articleNum=137274&clickid=zGZ2GG0YPQ0W18QXeqWmb2KAUkjUoBUQfTwvRM0&utm_campaign=Online%20Tracking%20Link&utm_medium=affiliate&utm_source=impactradius
      • Netgear -Â https://kb.netgear.com/13112/How-to-configure-Access-Control-or-MAC-Filtering-Smart-Wizard-routers
  • use a long, unique wifi password
  • use long, unique router admin username and password
  • don't leave default router credentials in place
  • update router firmware - consider using another OS like OpenBSD
  • use your own router; don't use a router provided by you ISP
  • don't use wifi, if possible
  • don't log in to your neighbor's or other open wifi networks
  • don't automatically join networks
  • don't leave credentials on post-it notes attached to your router
  • don't tell anyone what kind of router you use
  • consider setting up separate networks for guests and work
  • disable administration via the WAN interface and change the admin password.
    • there are very few people that need to administer via the WAN interface
    • if the WAN admin option is disabled, you won't be able to connect to the admin to even enter the username/password.
    • disabling wan stops most attacks but won't matter for some routers. Don't get those routers
    • once wan admin is turned off, the next attack vector is from unauthorized users that log in via wireless (if it's a wireless router)
    • just changing the password won't necessarily help. There could be bugs that allow you to bypass the authentication
  • disable wan pinging if that's an option
  • don't set up a dmz/direct ip:ip mapping. forward a few ports at most. that's usually all you need
  • remove /"forget" old wifi networks after they are no longer being used because you will expose your presence to those networks by automatically trying to connect every time you are within range.Â  CIA's "ELSA" uses wifi networks to geolocate targets:Â Â https://www.bleepingcomputer.com/news/security/vault-7-cia-malware-for-tracking-windows-devices-via-wifi-networks/

MiFi General Guidance for Travelers on the Go with No Access toÂ Hardwire

  • Travel with an ethernet cable and adapter appropriate to your computer.
  • If there is no traditional ethernet available do your best to locate a secure environment that has ethernet available.
  • If one is not available and communication is necessary, and you are utilizing a MiFi, consider the following:
    • your MiFi router should have an ethernet port and be able to get a wired 4G connection
    • you should ensure that a VPN is protecting your connection â€“ use a VPN that can be utilized to protect your phone and laptop(s) (NordVPN does this, you may also wish to check out ProtonVPN or another that can do this). Consult with Maximilian PowerÂ for questions regarding VPN usage.
    • do not hide the SSID, but you should certainly change the SSID (see below guidance underÂ Wifi)
    • Don't assume the connection is protected, and log off the MiFi whenever you are done using it.
    • do not give out credentials / do not share your MiFi with others.
    • If your phone can run Orbot, use it.
    • The above are general recommendations - take them with grains of salt. See additional hardening recommendations below.
